# 1\. What is Angular?

**Question:** _‚ÄúCan you explain what Angular is and why it‚Äôs used?‚Äù_

# 2\. What are Components in Angular?

**Question:** _‚ÄúWhat is a component in Angular, and how do you define one?‚Äù_

**Answer:** A **component** is the building block of any Angular app. It‚Äôs like a Lego block that makes up the entire application. Components are **reusable** and each one contains:

-   **HTML** for the view (template)
-   **CSS** for the style

Here‚Äôs a simple example of a component:

# 3\. What is Data Binding in Angular?

**Question:** _‚ÄúExplain the concept of data binding in Angular.‚Äù_

**Answer:** **Data binding** is how you connect your data (the model) to your view (HTML). Angular offers four types of data binding:

1.  **Interpolation**: Using `{{ }}` to bind data from the component to the view.

<p\>Hello, {{ name }}!</p\>

**2\. Property Binding**: Binding DOM properties to component values.

<img \[src\]="imageUrl" />

3\. **Event Binding**: Capturing user actions (like clicks) and binding them to methods in the component.

<button (click)="sayHello()"\>Click Me</button\>

4\. **Two-Way Binding**: Synchronizing data between the component and the view using the `[(ngModel)]` directive.

<input \[(ngModel)\]="name" />

**Humor Version:** ‚ÄúData binding in Angular is like mind reading, except instead of reading thoughts, you‚Äôre syncing your variables. One-way, two-way, all the ways! It‚Äôs like dating, but without the emotional baggage.‚Äù

# 4\. What is Dependency Injection (DI) in Angular?

**Question:** _‚ÄúWhat is dependency injection, and why does Angular use it?‚Äù_

**Answer:** **Dependency Injection (DI)** is a fancy term for **getting things you need** without having to create them manually every time. It‚Äôs like asking your butler to get you a coffee instead of brewing it yourself. In Angular, DI makes it easier to manage your services, classes, and objects. It also makes your code more testable by allowing you to inject mock data when testing.

Here‚Äôs an example of DI in Angular:

  providedIn: 'root',  
})

  getCoffee() {  
    return 'Here‚Äôs your coffee!';  
  }

  template: \`<p>{{ coffeeMessage }}</p>\`

  coffeeMessage: string;  
    
  constructor(private coffeeService: CoffeeService) {  
    this.coffeeMessage = this.coffeeService.getCoffee();

**Humor Version:** ‚ÄúDependency Injection is like Uber Eats for code ‚Äî when you need something, it gets delivered straight to your component. And you don‚Äôt even have to tip!‚Äù

# 5\. What are Directives in Angular?

**Question:** _‚ÄúWhat are Angular directives, and can you name a few?‚Äù_

**Answer:** Directives in Angular are like **instructions** you give to HTML elements. They can change the behavior or appearance of elements in your template. There are three types of directives in Angular:

1.  **Component Directives**: Components themselves are directives with a template.
2.  **Structural Directives**: Change the structure of the DOM (`*ngIf`, `*ngFor`)

<!--Before Control flow -->  
<p \*ngIf\="isLoggedIn"\>Welcome back!</p\>  
  
<!--Control flow -->  
@if(isLoggedIn){  
    <p\>Welcome back!</p\>  
}

3\. **Attribute Directives**: Modify the behavior or style of an element (`ngClass`, `ngStyle`).

<p \[ngClass\]="{ 'text-success': isLoggedIn }"\>Status: Logged In</p\>

**Humor Version:** ‚ÄúDirectives in Angular are like magic spells for your HTML. Want something to appear or disappear? Use `*ngIf`. Want to loop through your Hogwarts house members? Try `*ngFor`. Just make sure you‚Äôre not summoning any dark magic (or errors)."

# 6\. What are Angular Services?

**Question:** _‚ÄúWhat are services in Angular, and why are they used?‚Äù_

**Answer:** A **service** in Angular is a class that contains business logic or reusable functionality. Services are designed to be **shared** across components. They help in separating **concerns** and keeping the components focused on the UI.

Services are typically used for:

-   Fetching data from a server
-   Sharing data between components
-   Logging and authentication

Here‚Äôs a simple example of a service:

  providedIn: 'root'  
})

  isLoggedIn = false;  
    
  login() {  
    this.isLoggedIn = true;  
  }  
  
  logout() {  
    this.isLoggedIn = false;  
  }  
}

**Humor Version:** ‚ÄúServices in Angular are like that one friend who knows everything. Need data? Need help logging in? They‚Äôve got your back, and they‚Äôll let everyone know you‚Äôre logged in ‚Äî whether you like it or not.‚Äù

# 7\. What is Routing in Angular?

**Question:** _‚ÄúHow does routing work in Angular?‚Äù_

**Answer:** Angular **routing** allows you to navigate between different views (or pages) in a **single-page application** (SPA). Instead of loading a new page from the server, Angular dynamically swaps the views based on the route, making the application feel faster.

You configure routing in `app-routing.module.ts` by defining routes like this:

  { path: 'home', component: HomeComponent },  
  { path: 'about', component: AboutComponent },  
  { path: '', redirectTo: '/home', pathMatch: 'full' },

  exports: \[RouterModule\]

**Humor Version:** ‚ÄúRouting in Angular is like Google Maps for your app. Just define your routes and let Angular take you where you want to go ‚Äî without getting stuck in traffic (unless your code breaks).‚Äù
